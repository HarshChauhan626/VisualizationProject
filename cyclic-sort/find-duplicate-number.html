Find the Duplicate Number - Cyclic Sort

Problem Statement

Given an array of integers nums containing n + 1 integers where each integer is in the range [1, n] inclusive.

There is only one repeated number in nums, return this repeated number.

Example:
Input: nums = [1,3,4,2,2]
Output: 2

Constraints:
• You must solve the problem without modifying the array nums
• You must use only constant extra space

Difficulty: Medium
Pattern: Cyclic Sort / Floyd's Cycle Detection

Approach

Cyclic Sort (Modified Array)

Pseudo Code

// Cyclic Sort Approach (modifies array)
function findDuplicate(nums):
    i = 0
    while i < nums.length:
        if nums[i] != i + 1:
            j = nums[i] - 1  // correct position
            if nums[i] != nums[j]:
                swap(nums, i, j)
            else:
                return nums[i]  // found duplicate
        else:
            i++
    return -1
                
// Floyd's Cycle Detection (no modification)
function findDuplicate(nums):
    // Phase 1: Find intersection point in cycle
    slow = nums[0]
    fast = nums[0]
    do:
        slow = nums[slow]      // move 1 step
        fast = nums[nums[fast]] // move 2 steps
    while slow != fast
    
    // Phase 2: Find entrance to cycle
    slow = nums[0]
    while slow != fast:
        slow = nums[slow]  // move 1 step
        fast = nums[fast]  // move 1 step
    
    return slow  // duplicate number

Algorithm Details

Time Complexity

O(n) where n is the length of the array

Space Complexity

O(1) - constant extra space

How Cyclic Sort Works

1. Try to place each number at its correct position (number n at index n-1)
2. If we find a number that should go to a position already occupied by the same number, we found the duplicate
3. This approach modifies the input array

Time Complexity

O(n) where n is the length of the array

Space Complexity

O(1) - constant extra space

How Floyd's Algorithm Works

1. Treat the array as a linked list where arr[i] points to index arr[i]
2. Use slow and fast pointers to detect cycle
3. The duplicate number is the entrance to the cycle
4. This approach doesn't modify the input array
